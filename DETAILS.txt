--------------------------------------------------------------------------------
  1. WHAT IS BABYSAATHI?
--------------------------------------------------------------------------------

BabySaathi is a mobile app for Indian parents of babies aged 0 to 24 months.
It combines daily baby tracking (feed, sleep, diaper, growth, vaccines,
milestones) with deep analytics, an AI assistant that speaks in Hinglish like
a dadi/nani, Indian cultural content, and a community of other parents.

The app is built entirely offline-first — all tracker writes work without
internet and sync to Firebase when connectivity returns.

Target users  : Indian mothers and fathers, aged 22–35
Secondary     : Grandparents (Grandparent Mode with larger text + touch targets)
Pricing       : Free / Premium ₹199/mo / Family ₹349/mo (RevenueCat)
Languages     : 10 Indian languages (en, hi, bn, pa, ta, te, mr, gu, kn, ml)

--------------------------------------------------------------------------------
  8. SERVICES IN DETAIL
--------------------------------------------------------------------------------

notifications.ts
  requestNotificationPermissions()
  scheduleFeedReminder(babyName, lastFeedTime, avgIntervalMins)
    → fires at: lastFeedTime + avgInterval - 10 min
  scheduleSleepWindowAlert(babyName, lastWakeTime, wakeWindowMins)
    → fires at: lastWakeTime + wakeWindow - 15 min
  scheduleVaccineAlerts(babyName, vaccinations)
    → for each pending vaccine due within 14 days:
       day-before alert at 09:00, 3-days-before at 10:00
    → cancels existing vaccine alerts first; caps at 15 total

aiAssistant.ts
  buildSystemPrompt(baby, logs): string
    → Full dadi/nani persona system prompt
    → Baby profile block: name, age in weeks, gender, weight, blood group
    → Recent activity: timestamps, durations, feed types
    → "last feed >3hrs" warning when relevant
    → READY for real Claude API via Firebase Cloud Functions
  buildResponse(msg, baby, stats, logs): AIMessage
    → Local rule engine + regex intent routing
    → Date-fns for time calculations
    → References baby's actual name + real feed/sleep counts

feedAnalytics.ts
  fetchFeedAnalytics(baby, days: 7|14|30): Promise<FeedAnalyticsData>
    → Rejects invalid/future dates
    → Type breakdown: counts, %, avg durations, avg amounts
    → hourDistribution[24]: count per hour
    → Night hours: {22,23,0,1,2,3,4,5}
    → Longest stretches: top 3 gaps >60min
    → Cluster detection: 3+ feeds in any 3-hour window per day
    → Consolidation trend: first-half vs second-half daily average
      classified as consolidating|stable|increasing (±0.8 threshold)

growthPredictor.ts
  fetchGrowthTrendData(baby): Promise<GrowthTrendData | null>
    → Rejects null, future, pre-birth, implausible weights (>50 raw = grams)
    → Weight normalize: w > 50 ? w/1000 : w
    → Least-squares regression on last 6 points
    → Velocity: slope(kg/mo) × 1000 / 30.44 = g/day
    → WHO curve: P3/P50/P97 from getWHOCurveData(gender, 'weight', maxMonth)
    → Velocity bands: excellent/good/watch/low vs. VELOCITY_NORMS table
    → Forecasts: 4-week + 8-week with ±8%×(weeksAhead/4) confidence
    → Percentile trend: first vs. last of last-4 readings (+5pt = rising)

scheduleBuilder.ts + scheduleGuide.ts
  buildSchedule(baby): Promise<ScheduleResult>
    → 8 age bands: newborn(0-6w), 6-12w, 3-5m, 5-7m, 7-9m, 9-12m, 12-18m, 18-24m
    → Wake time: queries last 7 days of feeds, filters 4am-10am window,
      takes MEDIAN first-feed time; confidence: 'detected' | 'default'
    → Block types: wake/play/feed/nap/bedtime_routine/night_sleep/night_feed
    → 3+ nap schedules: uses 45-min catnap as final nap
    → Night feeds: evenly spaced by nightSleepDur/(nightFeeds+1)

vaccinePrep.ts
  fetchVaccinePrepData(baby): Promise<VaccinePrepData>
    → Guards corrupt scheduledDate with instanceof Date + isNaN
    → Paracetamol dose: mg = 10 × weightKg → ml = mg×5/120 → round ±0.25ml

contentModerator.ts
  autoFlag(content): FlagResult
    → 6 patterns: spam, unsafe_medical, crisis, abusive_language,
      misinformation, pii_phone
  reportPost(postId, reason, reporterUid, notes?)
    → Writes to /moderation_reports
    → Increments post.reportCount

offlineQueue.ts
  8 operation types: ADD_FEED, STOP_FEED, ADD_SLEEP, STOP_SLEEP,
                     ADD_DIAPER, ADD_GROWTH, MARK_VACCINE, MARK_MILESTONE
  Stores in AsyncStorage as QueuedOperation[]
  Flushes automatically via NetInfo (MAX_RETRIES = 3)
  init() / enqueue() / register() / flush() / destroy()

--------------------------------------------------------------------------------
  13. GETTING STARTED
--------------------------------------------------------------------------------

Prerequisites:
  Node.js 18+
  Expo Go app (quick testing) OR Expo Dev Build (subscriptions/notifications)
  Firebase project with Firestore, Authentication, Storage enabled
  RevenueCat account (subscription features only)

Install:
  cd BabySaathi
  npm install

Run:
  npm start            Expo Go
  npm run android      Android emulator / device
  npm run ios          iOS simulator / device

Config — Firebase (src/services/firebase/config.ts):
  Replace apiKey, authDomain, projectId, storageBucket, messagingSenderId, appId

Config — RevenueCat (src/constants/index.ts):
  REVENUECAT_API_KEY_IOS     = 'appl_YOUR_KEY'
  REVENUECAT_API_KEY_ANDROID = 'goog_YOUR_KEY'

Build for production:
  npm install -g eas-cli
  eas login && eas init
  eas build --platform android --profile production
  eas build --platform ios --profile production
  eas submit --platform android
  eas submit --platform ios
